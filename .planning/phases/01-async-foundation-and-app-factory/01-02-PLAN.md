<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → FastAPI route | Untrusted JSON body crosses here (already validated by Pydantic ChatRequest) |
| async route → asyncio.to_thread | Blocking pipeline executes in a threadpool thread; singletons shared across threads |
| lifespan → app.state | Singletons built at startup; readable by all concurrent requests; no mutation after startup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Denial of Service | async def chat() route | mitigate | await asyncio.to_thread() wraps all blocking FAISS/LLM/reranker calls; event loop is not blocked by pipeline |
| T-01-02-02 | Tampering | npm/pip/cargo installs | accept | No package installs in this plan; pytest and httpx already installed and audited in RESEARCH.md |
| T-01-02-03 | Elevation of Privilege | CORS middleware | mitigate | allow_methods narrowed to ["GET","POST"]; allow_headers narrowed to ["Content-Type"]; origins from settings.allowed_origins (not wildcard) |
| T-01-02-04 | Information Disclosure | /api/index/status | accept | Leaks index paths and model names; acceptable for Phase 1 (no auth yet); documented for Phase 3 auth gate |
| T-01-02-05 | Spoofing | LlmProvider singleton | accept | Singleton is constructed once in lifespan from settings.llm_provider; no request-time override possible without app restart |
| T-01-02-SC | Tampering | No new package installs | accept | Package legitimacy gate not applicable — no new packages introduced in this plan |
</threat_model>
